Step-by-Step Setup

1Register with Cerner Code Console

Visit: https://code.cerner.com

What you'll get:

  • Client ID
  • Client Secret
  • FHIR Base URL
  • OAuth Token URL
  • Tenant ID

2Configure iTechSmart HL7

Edit your .env file with Cerner credentials:

# Cerner Configuration CERNER_CLIENT_ID="your_cerner_client_id" CERNER_CLIENT_SECRET="your_cerner_client_secret" CERNER_FHIR_BASE_URL="https://fhir-myrecord.cerner.com/r4/your_tenant_id" CERNER_OAUTH_TOKEN_URL="https://authorization.cerner.com/tenants/your_tenant_id/protocols/oauth2/profiles/smart-v1/token" CERNER_TENANT_ID="your_tenant_id" # Optional: Cerner-specific settings CERNER_SYNC_INTERVAL="300" # Sync every 5 minutes CERNER_ENABLE_BULK_EXPORT="true" CERNER_MAX_RETRIES="3"

3Deploy iTechSmart HL7

Start the platform using Docker Compose:

# Navigate to project directory cd itechsmart-hl7 # Start all services docker-compose up -d # Check status docker-compose ps # View logs docker-compose logs -f backend

4Test Cerner Connection

Verify the integration is working:

# Test connection curl http://localhost:8000/api/emr/cerner/test-connection # Expected response: { "success": true, "message": "Cerner connection successful", "emr_system": "cerner", "fhir_version": "R4", "tenant_id": "your_tenant_id" }

5Fetch Patient Data

Start retrieving patient data from Cerner:

# Get patient by ID curl http://localhost:8000/api/patients/12345 # Get patient observations curl http://localhost:8000/api/patients/12345/observations # Get patient medications curl http://localhost:8000/api/patients/12345/medications # Get patient allergies curl http://localhost:8000/api/patients/12345/allergies
‚úÖ Success! Your Cerner integration is now active. Patient data will sync automatically every 5 minutes.

üîó Multi-EMR Setup Guide

üí° Why Multi-EMR? Many healthcare organizations use different EMR systems across departments, facilities, or acquired practices. iTechSmart HL7 provides a unified view across all systems.

Common Multi-EMR Scenarios

Scenario 1: Epic (Hospitals) + Cerner (Clinics)

Large health systems often use Epic for inpatient care and Cerner for outpatient clinics.

# Configure both systems in .env # Epic Configuration EPIC_CLIENT_ID="your_epic_client_id" EPIC_CLIENT_SECRET="your_epic_secret" EPIC_FHIR_BASE_URL="https://fhir.epic.com/interconnect-fhir-oauth/api/FHIR/R4" # Cerner Configuration CERNER_CLIENT_ID="your_cerner_client_id" CERNER_CLIENT_SECRET="your_cerner_secret" CERNER_FHIR_BASE_URL="https://fhir-myrecord.cerner.com/r4/your_tenant" # Enable both integrations ENABLE_EPIC="true" ENABLE_CERNER="true"

Scenario 2: Epic + Cerner + Meditech

Complex health systems with multiple acquisitions may have three or more EMR systems.

# Epic Configuration EPIC_CLIENT_ID="your_epic_client_id" EPIC_FHIR_BASE_URL="https://fhir.epic.com/..." # Cerner Configuration CERNER_CLIENT_ID="your_cerner_client_id" CERNER_FHIR_BASE_URL="https://fhir-myrecord.cerner.com/..." # Meditech Configuration MEDITECH_CLIENT_ID="your_meditech_client_id" MEDITECH_FHIR_BASE_URL="https://fhir.meditech.com/..." MEDITECH_HL7_HOST="hl7.meditech.com" MEDITECH_HL7_PORT="2575" # Enable all integrations ENABLE_EPIC="true" ENABLE_CERNER="true" ENABLE_MEDITECH="true"

Setup Steps for Multi-EMR

1Register with Each EMR Vendor

  • Epic: Register at apporchard.epic.com
  • Cerner: Register at code.cerner.com
  • Meditech: Contact Meditech support for API credentials
  • Allscripts: Contact Allscripts for Unity API access

2Configure All Systems

Create a comprehensive .env file with all credentials:

# ======================================== # iTechSmart HL7 - Multi-EMR Configuration # ======================================== # Epic Configuration ENABLE_EPIC="true" EPIC_CLIENT_ID="your_epic_client_id" EPIC_CLIENT_SECRET="your_epic_secret" EPIC_FHIR_BASE_URL="https://fhir.epic.com/interconnect-fhir-oauth/api/FHIR/R4" EPIC_OAUTH_TOKEN_URL="https://fhir.epic.com/interconnect-fhir-oauth/oauth2/token" # Cerner Configuration ENABLE_CERNER="true" CERNER_CLIENT_ID="your_cerner_client_id" CERNER_CLIENT_SECRET="your_cerner_secret" CERNER_FHIR_BASE_URL="https://fhir-myrecord.cerner.com/r4/your_tenant" CERNER_TENANT_ID="your_tenant_id" # Meditech Configuration ENABLE_MEDITECH="true" MEDITECH_CLIENT_ID="your_meditech_client_id" MEDITECH_FHIR_BASE_URL="https://fhir.meditech.com/api/FHIR/R4" MEDITECH_HL7_HOST="hl7.meditech.com" MEDITECH_HL7_PORT="2575" # Allscripts Configuration ENABLE_ALLSCRIPTS="true" ALLSCRIPTS_APP_NAME="your_app_name" ALLSCRIPTS_USERNAME="your_username" ALLSCRIPTS_PASSWORD="your_password" # Sync Settings SYNC_INTERVAL="300" # 5 minutes ENABLE_REAL_TIME_SYNC="true" MAX_CONCURRENT_SYNCS="5"

3Deploy and Test

Option 1: HL7 v2.x Custom Integration

1Configure HL7 Connection

# Configure custom HL7 connection in .env CUSTOM_HL7_ENABLED="true" CUSTOM_HL7_HOST="your-hl7-server.com" CUSTOM_HL7_PORT="2575" CUSTOM_HL7_VERSION="2.5" CUSTOM_HL7_ENCODING="UTF-8" # Message types to process CUSTOM_HL7_MESSAGE_TYPES="ADT,ORM,ORU,MDM" # Custom segment handling CUSTOM_HL7_CUSTOM_SEGMENTS="ZPD,ZRX,ZDG"

2Create Custom Parser (Optional)

For custom segments or non-standard HL7 implementations:

# Create custom_parser.py in backend/app/integrations/ from hl7apy.parser import parse_message class CustomHL7Parser: def parse_custom_message(self, hl7_message): """Parse custom HL7 message with proprietary segments""" msg = parse_message(hl7_message) # Extract standard segments patient_id = msg.pid.pid_3.value patient_name = msg.pid.pid_5.value # Extract custom ZPD segment (custom patient data) if hasattr(msg, 'zpd'): custom_field_1 = msg.zpd.zpd_1.value custom_field_2 = msg.zpd.zpd_2.value return { 'patient_id': patient_id, 'patient_name': patient_name, 'custom_data': { 'field_1': custom_field_1, 'field_2': custom_field_2 } }

Option 2: Custom REST API Integration

1Configure API Connection

# Configure custom API in .env CUSTOM_API_ENABLED="true" CUSTOM_API_BASE_URL="https://api.youremr.com/v1" CUSTOM_API_AUTH_TYPE="bearer" # or "basic", "api_key" CUSTOM_API_TOKEN="your_api_token" # Endpoint mappings CUSTOM_API_PATIENT_ENDPOINT="/patients/{id}" CUSTOM_API_OBSERVATIONS_ENDPOINT="/patients/{id}/observations" CUSTOM_API_MEDICATIONS_ENDPOINT="/patients/{id}/medications"

2Create Custom Integration Module

# Create custom_api_integration.py import requests from typing import Dict, Any class CustomAPIIntegration: def __init__(self, base_url: str, token: str): self.base_url = base_url self.headers = { 'Authorization': f'Bearer {token}', 'Content-Type': 'application/json' } def get_patient(self, patient_id: str) -> Dict[str, Any]: """Fetch patient data from custom API""" url = f'{self.base_url}/patients/{patient_id}' response = requests.get(url, headers=self.headers) if response.status_code == 200: data = response.json() # Transform to iTechSmart format return { 'patient_id': data['id'], 'name': data['full_name'], 'date_of_birth': data['dob'], 'gender': data['sex'], 'source': 'custom_api' } return None

Option 3: File-Based Integration

1Configure File Import

# Configure file-based integration FILE_IMPORT_ENABLED="true" FILE_IMPORT_DIRECTORY="/data/imports" FILE_IMPORT_SCHEDULE="0 */6 * * *" # Every 6 hours # Supported formats FILE_IMPORT_FORMATS="csv,xlsx,hl7,json" # FTP/SFTP settings (optional) FTP_ENABLED="true" FTP_HOST="ftp.youremr.com" FTP_USERNAME="your_username" FTP_PASSWORD="your_password" FTP_REMOTE_PATH="/exports"

2Define File Format Mapping

Create a mapping file for CSV/Excel imports:

# Create file_mapping.json { "patient_import": { "file_pattern": "patients_*.csv", "columns": { "MRN": "patient_id", "FirstName": "first_name", "LastName": "last_name", "DOB": "date_of_birth", "Sex": "gender" }, "date_format": "%Y-%m-%d" } }
